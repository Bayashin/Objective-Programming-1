Javaでのマルチスレッドプログラミング

プログラミング言語Javaの特徴の1つとして、マルチスレッドの標準APIによるサポートが挙げられます。

スレッドとは、プログラムを実行する1つの主体を指します。マルチスレッドとは、1つのプログラムから、処理を分割したプログラムの実行を複数の主体に任せることができる機能のようなもので捉えてもらうと良いでしょう。
(たとえ話をするなら、複数人が集まって行うグループワークにおいて、リーダーの指示で各メンバーに作業を行わせるイメージがもっとも近いように思います。)

マルチスレッド用のプログラム作成方法

Javaにおけるマルチスレッド用プログラムの作成方法には、代表的な2つの方法があります。

  1. Threadクラスを継承したクラスを作成する
  2. Runnableインターフェイスを実装したクラスを作成する

どちらの方法も、public void run()メソッドをオーバーライドすることで各スレッドが実行されたときに呼び出される処理を記述できます。

継承とインターフェイスの実装と2種類の方法の違いは、呼び出し方法の違いのみで大きく異なる点はほぼありません。

Threadクラスを継承する場合は、Javaの言語仕様として多重継承できない点が重要になります。他のクラスを継承した上でマルチスレッドに対応させるためには、必ずRunnableインターフェイスを実装する方法しかありません。 （昨今の匿名クラスやラムダの記法を用いるならば、Runnableインターフェイスを用いる方法をオススメします。）

スレッドの実行方法

スレッドを実行させる場合には、上記の2パターンでそれぞれ呼び出しかたが異なります。

1. Threadクラスを継承したクラスを作成した場合

Threadを継承したクラス自体をインスタンス化し、run()メソッドではなく、start()メソッドを呼ぶことでスレッドを実行できます。

class ThreadSample extends Thread {
  @Override
  public void run() {
    // なにかしらの処理
    System.out.println("スレッドが呼び出されました");
  }

  public static void main(String[] args) {
    ThreadSample t1 = new ThreadSample();
    ThreadSample t2 = new ThreadSample();
    ThreadSample t3 = new ThreadSample();
    t1.start();  //スレッドの実行
    t2.start();  //スレッドの実行
    t3.start();  //スレッドの実行
  }
}

2. Runnableインターフェイスを実装したクラスを作成する

Threadクラス自体のインスタンス生成時の引数に、Runnableインターフェイスを実装したクラスのインスタンスを渡すことでスレッドを作成、start()メソッドを呼ぶことでスレッドを実行できます。

class RunnableSample implements Runnable {
  @Override
  public void run() {
    // なにかしらの処理
    System.out.println("Runnableなスレッドが呼び出されました");
  }

  public static void main(String[] args) {
    Thread t1 = new Thread(new RunnableSample());
    Thread t2 = new Thread(new RunnableSample());
    Thread t3 = new Thread(new RunnableSample());
    t1.start();  //スレッドの実行
    t2.start();  //スレッドの実行
    t3.start();  //スレッドの実行
  }
}

手順が1つ増えますが、こちらの手法はインターフェイスの実装で行えるため、既存の継承がある場合などにはこの手法が使えます。

public class Athlete {

  /**
   * 速度[m/s]を指定して初期化
   * @param name
   * @param speed
   */
  public Athlete(String name, double speed) {
    this.name = name;
    this.speed = speed;
  }

  private final double speed;
  private final String name;

  /**
   * 選手名
   */
  public String getName() {
    return name;
  }

  /**
   * 選手の走る速度[m/s]
   */

    return speed;
  }
}

走行レーンクラス Lane.java

import java.text.DecimalFormat;

public class Lane {
  private final Athlete athlete;
  private final int runningDistance;
  private double time;

  public Athlete getAthlete() {
    return athlete;
  }

  public Lane(Athlete athlete, int runningDistance) {
    this.athlete = athlete;
    this.runningDistance = runningDistance;
  }

  public double getTime() {
    return time;
  }

  public void run() {
    DecimalFormat df = new DecimalFormat("0.00");

    System.out.println(athlete.getName() + ":スタート(速度: " + df.format(athlete.getSpeed()) + "[m/s])");

    // 走る時間を計算
    double t = runningDistance / athlete.getSpeed();
    long mills = Math.round(t * 1000);

    try {
      // スレッドを止めて走っている演出
      Thread.sleep(mills);
    } catch(InterruptedException e) {
    }

    // タイムを設定
    this.time = t;

    System.out.println(athlete.getName() + ":ゴール タイム:" + df.format(t) + "[s]");
  }
}

徒競走クラス Footrace.java

import java.util.ArrayList;
import java.util.Collections;
import java.util.List;

public class Footrace {
  public static final int RUNNING_DISTANCE = 30;

  private final List<Athlete> athletes = new ArrayList<>();

  public List<Athlete> getAthletes() {
    return athletes;
  }

  public void start() {

    System.out.println(RUNNING_DISTANCE + "メートル走");

    // 走行レーンの設定
    List<Lane> lanes = new ArrayList<>();
    for (Athlete ath : athletes) {
      lanes.add(new Lane(ath, RUNNING_DISTANCE));
    }

    // 選手一人ずつ走る
    for (Lane l : lanes){
      l.run();
    }

    // タイムでソート
    Collections.sort(lanes, (Lane a, Lane b) -> a.getTime() == b.getTime() ? 0 : (a.getTime() < b.getTime() ? -1 : +1));

    System.out.println("=== 結果発表 ===");
    int rank = 1;
    for (Lane l : lanes){
      System.out.printf("第%d位\t%s\t%.2f[s]%s", rank++, l.getAthlete().getName(), l.getTime(), System.lineSeparator());
    }

  }
}

StreamAPIを用いたマルチスレッド化

Java8より追加されたStreamAPIのparallelStreamを用いると、ラムダ記法を用いて並列処理を行えます。

// Footrace.javaの変更のみでよいです。

lanes.parallelStream().forEach(l -> {

この書き方は、もうすでにThreadに処理を任せたいものがすべて出揃っている場合にのみ使用できる書き方です。

マルチスレッドプログラミングの注意点

Javaでは、標準的にサポートされるThread関連クラスにより、簡単にマルチスレッドプログラミングが可能です。

しかし、注意点も多数存在します。

たとえば、複数のスレッドから同時に1つのstaticな変数を操作した場合や、ファイルにアクセス（主に書き込み処理）を行った場合どうなるかを考慮しなければなりません。(主にデッドロックなどが発生しやすくなります。)

これは、複雑な処理をマルチスレッド化する際、気づかないうちにやってしまうミスとして挙げられます。

マルチスレッド化を行う場合には、「極力1つ1つのスレッドが他のスレッドのオブジェクトを参照しないようにする」、「スレッドセーフなコードを用いる」などのポイントを押さえておくとよいでしょう。

講義内では時間の都合上、これ以上掘り下げた解説は行いません。 詳しくは、各自で書籍を買うなり、インターネットで調べるなりしてベストプラクティスを探してみるとよいでしょう。

java.util.concurrent.ExecutorService を使ったThreadPoolの活用

上記で紹介したThreadの方法では、無尽蔵にThread生成ができてしまう点と、そもそもThread自体起動に若干の時間がかかってしまう点が問題としてあります。

Javaには、java.util.concurrent.ExecutorServiceなどのExecutor関連のThread処理機構が存在します。

以下のコードでは、「Threadの名前と開始の文字列出力→3秒待機→Threadの名前と終了の文字列出力」という簡単なRunnableな処理を作成し、それを所定の回数ThreadPoolに格納しつつ処理を行う内容となっています。

ThreadPoolとは、複数のThreadをあらかじめ作成して待機させておき、実行するべき処理が来たら待機させたThreadの処理待ち行列に処理を割り当てる仕組みのことをいいます。こうした仕組みは自前で実装する必要はなく、ExecutorServiceを使うことで簡単に利用できます。

import java.util.concurrent.ExecutorService ;
import java.util.concurrent.Executors;
import java.util.concurrent.TimeUnit;
import java.util.stream.IntStream;

public class Main {

    public static void main(String[] args) {

        // Runnableな処理を作成
        Runnable command = () -> {
            System.out.println(Thread.currentThread().getName() + ":Start");
            try {
                Thread.sleep(3000);     // 3秒待つ
            } catch (InterruptedException e) {
                throw new RuntimeException(e);
            }
            System.out.println(Thread.currentThread().getName() + ":End");
        };

        // ThreadPoolを3つ用意しておく
        ExecutorService executor = Executors.newFixedThreadPool(3);
        try {
            // 10個Runnableな処理をThreadPoolに入れつつ実行
            IntStream.range(0, 10).forEach(i -> {
                executor.execute(command);
            });
        } finally {
            // shutdownを呼ぶことで、ThreadPoolを開放し、タスクの追加が行えなくなる
            executor.shutdown();
            try {
                //シャットダウン要求後、タスクが実行を完了するか、タイムアウトするか、割り込みなどで停止するかまで待つ
                executor.awaitTermination(1, TimeUnit.MINUTES);
            } catch (InterruptedException e) {
                throw new RuntimeException(e);
            }
        }
    }
}

この書き方であれば、Threadの領域確保にかかる時間をThreadPoolを3つ分確保するだけになり、処理の高速化ができ、Threadの生成上限を3つに絞ることができリソースの節約が可能となります。

Webアプリケーション構築の場合や、大量なThreadが生成されることを予想されるプログラムを作成しようとしている方は、こちらのExecutorServiceを使うと良いでしょう。
